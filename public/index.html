About the project

Busin 0 is a 70+ hour Wizardry-style first-person dungeon crawler that never left Japan. This project brings it into English by patching the actual disc data, so it runs on real PS2 hardware — not just emulators.

  • 🗺

    A lost JP-only crawler

    A 70+ hour first-person dungeon crawler in the Wizardry tradition, now playable in English.

  • ♜

    Real PS2 compatible

    The patch modifies the real disc data — fonts, scripts, menus — so it works on hardware, not just emulator overlays.

  • ✍

    ~83,000 lines translated

    Dialogue, narration, menus and item text translated across the entire game.

  • ⚙

    ~13,000 lines of tooling

    Custom reverse-engineering tools were written from scratch to decode the game's formats and re-inject English.

  • ⚔

    Fully localized systems

    Character creation, combat, quests, equipment and menus are all in English.

  • 🛡

    A living public beta

    The major systems work start to finish; bug reports from players drive the project forward.

How to apply

  1. Verify your JP ISO's MD5 matches the source hash below.
  2. Apply with xdelta3 on the command line: 
    xdelta3 -d -s "Busin 0 (Japan) SLPM-65378.iso" busin0_en_v191.xdelta BUSIN0_EN.iso
  3. Or use a GUI such as Delta Patcher: set the original file to your JP ISO, the patch to busin0_en_v191.xdelta, then Apply.
  4. Boot the resulting ISO fresh from the title screen. Do not load an old save state from before patching.

v190 July 2026 The dialogue loops that could freeze the game are fixed, item names fit their menus, and the Janken tell works again.
  • The dialogue loops that could freeze the game are fixed. On certain floors, some conversations — the shop-creation scene on B1, the soldier after the B8 Undead Dragon fight, and the Ingo scene on B10 — could fall out of sync and loop forever, softlocking the game (issues #27/#30/#31). The cause was ours: speaker-labeled event dialogue in a part of the script our tools couldn’t reach was left pointing at the wrong text once English made lines longer, so the game read past the end and looped. We made a general fix that repairs this across the whole game — every floor, roughly 160 spots — not just the three reported. Because it’s a broad structural change, it needs verification: if any conversation still loops or shows a garbled speaker name, please let us know. (Minor cosmetic note: a speaker’s name may briefly appear at the start of their line — being polished.)
  • Item names now fit their menus. Long English names like “Faith Hide Armor” or “Resurrection Potion” overran their slots on every inventory screen, drawing over your stats and item counts. All 228 over-long names were shortened to fit (for example Resurrection Potion → Raise Potion, Town Return Potion → Return Potion, Blood-Stained Knife → Bloody Knife), so the bag, shop, and battle menus read cleanly.
  • The Janken Man’s tell works again. The rock-paper-scissors minigame has a visual “tell” — a trembling character that telegraphs the opponent’s next throw — that had been lost in translation. It’s restored, so you can read it and win the prizes (issue #26).
  • Huge thanks to ferrary-neto, Salomae, Gyoru, and SirBewm for the reports and save files. This disc is SLPM-65378, software v1.03.

v189 July 2026 Deep-floor battles that froze the moment they began are fixed.
  • Battles that froze the game are fixed. On the lower floors, starting a fight against certain enemies could freeze the game the instant the battle began. The cause was ours: a cosmetic banner tweak had blindly overwritten a block of the game’s own battle-setup data, which starved the graphics memory and locked the fight up (and, in one build, made monsters invisible and blocked fleeing). That change has been removed entirely — it never served any purpose, because the banner it aimed at is drawn a different way. The B5 spider fights are confirmed fixed (monsters visible, fight and flee both work). The other flying-shadow encounters reported crashing — gargoyle, crawling shadow, wisp — share the exact same cause and should be fixed as well, but we haven’t re-tested each one individually yet, so if any still gives you trouble, please let us know.
  • A round of small wording fixes in battle and menus. In the item menu the identify option now reads “ID” (was “Rcv”) and discard reads “Drop”; the identify messages read “[name] IDing” and “Success.” The end-of-turn prompts are “Yes / No / Confirm” (were “Go!!/Redo/Results”), and the little spider’s web attack reads “Web” (was “Hit”). One ally-action line that ran past its box (“…all mem”) now fits.
  • Huge thanks to ferrary-neto and Salomae — the crash reports, and especially the save files, are what made it possible to reproduce the freeze and trace it to the root. This is the project’s biggest stability fix so far. This disc is SLPM-65378, software v1.03.

Report bugs

This is a public beta — bug reports are the whole point. Deep and late-game content is largely untested.

When something breaks, the most useful thing you can send is a save to reproduce from. Please include:

  • A PCSX2 save state (default hotkey F1) from just before a crash — not at the frozen screen.
  • An in-game memory-card save near the problem.
  • A one-line description of what you did.
  • A screenshot for any bad text, formatting or untranslated content.
  • The patch version (v180 beta).
  • Your emulator and version (or "real PS2"), plus your PCSX2 renderer (Hardware vs Software) for graphics glitches.

The best place for bug reports is GitHub Issues — you can attach save states and screenshots directly (zip the .p2s first). No GitHub account? Reply in the r/wizardry release thread instead.
